# 全传感器类型串口协议分析

## 通用协议

所有传感器共享相同的分隔符：`AA 55 03 99`（4字节），使用 DelimiterParser 解析。

## 帧结构

所有帧以 `AA 55 03 99` 分隔，帧体紧跟分隔符后。

### 1024B 帧（坐垫/矩阵数据）
- 1024 字节 = 32×32 矩阵
- 每字节为一个压力值（0-255）
- 不同 file 类型有不同的线序变换函数

### 130B 帧（手部蓝牙第一帧）
- byte[0]: order（顺序位）
- byte[1]: type（类型位）
- byte[2-129]: 128字节数据

### 146B 帧（手部蓝牙第二帧）
- byte[0]: order
- byte[1]: type
- byte[2-129]: 128字节数据
- byte[130-145]: 16字节四元数数据

### 256B 帧（16×16矩阵）
- 256 字节 = 16×16 矩阵

### 4096B 帧（64×64矩阵）
- 4096 字节 = 64×64 矩阵

### 142B + 158B 帧（daliegu 14×20）
- 142B: byte[0]=order, byte[1]=type, byte[2-141]=140字节数据
- 158B: byte[0]=order, byte[1]=type, byte[2-141]=140字节数据, byte[142-157]=16字节四元数
- 合并后 280 字节 = 14×20 矩阵
