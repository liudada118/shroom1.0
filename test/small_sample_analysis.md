# 小型样品传感器协议分析

## 原始数据解析

```
AA 55 03 99  ← 分隔符
01 06        ← order=1(第一帧), type=6(新类型)
00 00 ... (128字节数据，共130字节帧体)
AA 55 03 99  ← 分隔符
02 06        ← order=2(第二帧), type=6(新类型)
00 00 ... (128字节数据)
F1 FF 7F 3F E0 B6 2F 3A F0 F0 08 BA 88 7C 84 BA  ← 16字节四元数
AA 55 03 99  ← 下一帧分隔符
```

## 帧结构（与hand0205相同）

- 第一帧: 130字节 = 1(order) + 1(type) + 128(数据)
- 第二帧: 146字节 = 1(order) + 1(type) + 128(数据) + 16(四元数)
- 合并后: 256字节有效数据 (128+128)
- type=6 是新的设备类型标识

## 输出要求

- 10×10 数字矩阵 = 100个数据点
- 从256字节中取前100个（或按特定映射）
- 显示形式：类似32×32高速的数字网格
