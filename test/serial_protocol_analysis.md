# 串口协议分析

## 帧分隔符
AA 55 03 99 (0xAA, 0x55, 0x03, 0x99)

## 波特率
默认: 1000000 (1Mbps)
可选: 921600, 3000000

## 串口角色
- port1 → parser (坐垫/主传感器) → WS 19999
- port2 → parser2 (靠背) → WS 19998
- portHead → parser4 (头枕) → WS 19997
- port1 → parser3 (大床模式备用)

## 帧长度与数据类型
| 帧长度 | 数据类型 | 说明 |
|--------|----------|------|
| 1024 字节 | 矩阵数据 | 32x32=1024 压力传感器矩阵，每字节一个点(uint8) |
| 72 字节 | 小矩阵 | 小型传感器 |
| 144 字节 | 中型矩阵 | 中型传感器 |
| 130 字节 | 手部第一帧 | [顺序位(1)][类型位(1)][数据(128)] |
| 146 字节 | 手部第二帧 | [顺序位(1)][类型位(1)][数据(128)][四元数(16)] |
| 262 字节 | 手套数据 | [数据(256)][旋转(6)] |
| 1025 字节 | 大床数据 | [数据(1024)][帧序号(1)] |

## 默认传感器类型
file = 'hand0205' (手部传感器)

## WebSocket 广播数据格式
主串口(sitData): { sitData: [...], hz: colHZ, sitFlag: bool, backFlag: bool }
靠背(backData): { backData: [...], sitFlag: bool, backFlag: bool }
头枕(headData): { headData: [...] }
手部: { sitData: [...], rotate: [...], newArr147: [...], realArr: [...] }

## 测试模拟策略
1. 用 socat 创建虚拟串口对
2. 模拟 1024 字节帧 (32x32 矩阵) 用于大多数传感器类型
3. 模拟 130+146 字节帧对 用于手部传感器 (hand0205)
4. 每帧前加 AA 55 03 99 分隔符
