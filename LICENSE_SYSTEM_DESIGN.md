# 新密钥授权系统设计方案

## 核心目标
- 单密钥单设备绑定，不可跨机器使用
- 不容易被破解（RSA 非对称签名，私钥不在客户端）
- 支持离线和在线两种模式
- 时间回拨检测

## 架构概览

### 机器码（Machine ID）
- 采集：MAC 地址 + 硬盘序列号 + CPU ID
- 用 SHA-256 哈希生成 16 位短码（方便客户抄写）
- 模块：`machineId.js`

### 离线模式
1. 客户在密钥页面点击"获取机器码"→ 显示 16 位机器码
2. 客户把机器码发给管理员
3. 管理员用 License 生成工具（私钥签名）生成激活码
4. 激活码内容：{ machineId, sensorTypes, expireDate, timestamp }
5. 客户输入激活码 → 软件用内置公钥验证签名 → 校验机器码匹配 → 激活

### 在线模式
1. 软件发送机器码到服务端
2. 服务端返回授权信息（已签名）
3. 本地用公钥验证 → 激活

### 时间回拨检测
- 每次验证成功后，记录当前时间戳到本地加密文件
- 下次启动时比较：如果当前时间 < 上次记录时间，判定为回拨
- 回拨后锁定软件，需重新联网验证

### 文件结构
- `machineId.js` - 机器码采集
- `timeGuard.js` - 时间回拨检测
- `licenseV2.js` - 新授权核心（RSA 验签、激活码解析）
- `licenseKeygen.js` - 激活码生成工具（管理员用，含私钥）
- `keys/public.pem` - 内置公钥（随软件分发）
- `keys/private.pem` - 私钥（仅管理员持有，不随软件分发）
- `license.dat` - 本地授权数据（加密存储）
- `timeguard.dat` - 时间戳记录（加密存储）
